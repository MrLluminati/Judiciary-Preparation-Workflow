# Month 1 Rigorous Study Schedule

## Period Covered

**14.05.2026 to 13.06.2026**

## Operating Rule From 14.05.2026 Onwards

At the start of every study day, the day must be divided into planned blocks in advance.

For every study block, record:

1. **Original Plan** — what was scheduled before any backlog was added.
2. **Backlog Added** — what was shifted into the block from an earlier block/day.
3. **Actual Completion** — what was actually completed.
4. **Carry Forward** — what remains pending after the block.

This is mandatory so that progress can be measured honestly against the target timeline.

## Daily Block Structure

### Court Days / Working Days

| Block | Time Nature | Purpose |
|---|---|---|
| Study Block 1 | Morning / before court | Recall + bare act reading + first concept |
| Study Block 2 | Evening / after court | Main concept completion + drills |
| Study Block 3 | Night / light block | MCQs, revision, error-log, dashboard and documentation updates |

### Sunday / Low-Court-Pressure Days

| Block | Purpose |
|---|---|
| Block 1 | Weekly revision + pending backlog clearance |
| Block 2 | Mock / PYQ / mains skeleton writing |
| Block 3 | Dashboard, notebook cleanup, next week planning |

## Core Strategy

1. Complete one bare act/module at a time.
2. Do daily recall before new clauses.
3. Do short drills after every concept.
4. Do MCQs after each article cluster.
5. Do mains skeletons after each major theme.
6. Update GitHub and Google Sheet only at study-block closure, not during active study.

# Measurement Rules

## Daily Success

A day is successful if:

1. At least two study blocks are completed; and
2. all completed blocks have been logged; and
3. backlog is expressly marked, not hidden.

## Weekly Success

A week is successful if:

1. one major topic cluster is completed;
2. at least one MCQ set is attempted;
3. at least one mains skeleton is attempted;
4. error log is updated.

## Backlog Rule

Backlog may be moved only once automatically. If the same backlog survives two blocks, it must be placed in Sunday backlog clearance.

## Documentation Rule

At the end of every completed study block:

1. Update progress tracker.
2. Update relevant notebook/practice/error-log if needed.
3. Update Google Sheet dashboard.
4. Mark original plan vs backlog added.
