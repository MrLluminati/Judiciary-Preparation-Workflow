## Mandatory Block Tracking Rule

At the start of every study day, record the original plan for each block before adding backlog.

For each block, maintain:

| Field | Meaning |
|---|---|
| Original Plan | What the block was scheduled to cover originally |
| Backlog Added | What was shifted into this block from an earlier block/day |
| Actual Completion | What was actually completed |
| Carry Forward | What remains pending |
| Status | Not Started / In Progress / Closed / Carried Forward |

This is mandatory because it shows whether progress is on schedule or backlog-driven.

# Measurement Rules

## Daily Success

A day is successful if:

1. At least two study blocks are completed.
2. Original plan vs backlog added is recorded.
3. One MCQ set or one answer-writing output is completed.
4. Backlog is expressly marked, not hidden.

## Weekly Success

A week is successful if:

1. the wall-planner subject milestone is substantially completed;
2. one MCQ/PYQ set is attempted;
3. one mains skeleton is attempted;
4. error log and dashboard are updated.

## Backlog Rule

Backlog may be moved only once automatically. If the same backlog survives two blocks, it must be placed into Sunday backlog clearance.

## Dashboard Rule

The Google Sheet dashboard must show:

1. 12-week roadmap milestone status;
2. Month 1 milestones;
3. current week status;
4. block-level original plan vs backlog;
5. progress charts.
